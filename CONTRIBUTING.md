# Contributing

## Before Opening a PR

- keep changes focused
- preserve working links and image paths
- keep profile copy factual and concise
- avoid adding trackers, heavy embeds, or unnecessary badges

## Local Checks

- preview the Markdown on GitHub or in a Markdown viewer
- verify profile links still resolve
- keep images optimized and referenced from tracked files only

## PR Notes

Include:
- what changed
- why it changed
- how you verified it
